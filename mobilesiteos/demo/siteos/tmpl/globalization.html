Globalization

This is a demo of the jQuery Globalize Library. You can view the source or keep up to date with issues, changes, etc. for this project on GitHub: Globalize on GitHub
As demonstrated in our version called 'msos.intl', alternate culture calendars are built by extending the culture's standard calendar with the specific alternate calendar's different parameters. Not entirely sure that this is correct (what the db of cultures was meant to represent), so be fore warned and/or email corrections. As of this writing, the globalize.js demo page seems to just ingore the missing parameters.
The other important difference in this implementation of the jQuery Globalize Library is that only the files needed for each language/culture (locale) are loaded. This means a few more http ajax calls, but the user's device only loads the needed, individulally small, culture specific files.

Interesting Cultures to Preview

ja-JP
Japanese (Japan):
Has an optional calendar that has 4 different eras.
th-TH
Thai (Thailand):
Standard calendar has a year offset.
te-IN
Telugu (India):
Numbers have unequal groupings.
ar-SA
Arabic (Saudi Arabia):
Standard calendar is UmAlQura, a non-gregorian based calendar that requires custom conversion logic. It also supports the Hijri calendar, and a standard gregorian calendar translated into one of 4 languages.
